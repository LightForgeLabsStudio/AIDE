# Language-Specific Examples

This directory contains complete, ready-to-use examples of AIDE documentation for different technology stacks.

## Available Examples

### [godot-gdscript/](godot-gdscript/)
**Game development with Godot Engine**

- GUT testing framework
- GDScript style guide
- Scene-based architecture
- Tested on: Lightborn Exile (2D roguelike factory game)

### [nodejs-typescript/](nodejs-typescript/)
**Web applications and backend services**

- Jest + Supertest testing
- TypeScript + Express/Fastify
- Clean architecture patterns
- Suitable for: REST APIs, web apps, microservices

### [python/](python/)
**Backend services, ML, and scripting**

- pytest testing framework
- FastAPI / Django patterns
- PEP 8 style guide
- Suitable for: APIs, data processing, ML services

### [rust/](rust/)
**Systems programming and high-performance services**

- cargo test framework
- Actix / Axum patterns
- Rust idioms and error handling
- Suitable for: Performance-critical services, CLI tools

## How to Use These Examples

1. **Choose your stack** - Browse the example closest to your project
2. **Copy the files** - Copy `TESTING_POLICY.md`, `CODING_GUIDELINES.md`, etc. to your project
3. **Customize** - Replace any remaining `{{PLACEHOLDERS}}` with your project details
4. **Adopt patterns** - Use the examples as a starting point; adapt to your needs
